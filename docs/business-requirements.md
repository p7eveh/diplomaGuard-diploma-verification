# Бизнес-требования DiplomaGuard

## 1. Бизнес-цели (со стороны заказчика / владельца продукта)

| Цель | Показатель достижения | Критерий успеха |
|------|----------------------|----------------|
| Снизить количество поддельных дипломов | % работодателей, столкнувшихся с подделкой | Снижение на 80% среди клиентов системы |
| Ускорить проверку дипломов | Время от запроса до результата | Менее 15 секунд |
| Уменьшить затраты вузов на выдачу дипломов | Стоимость выпуска одного диплома | < $0.50 |
| Сделать проверку доступной для всех | Отсутствие регистрации для HR | 100% проверок без входа в систему |

## 2. Критерии достижения бизнес-целей

- Система должна обрабатывать минимум 1000 запросов верификации в день
- Время отклика API при проверке < 2 секунд
- Стоимость газа на выпуск диплома не более 200 000 gas
- Интеграция с MetaMask только для эмитентов (вузов)

## 3. Бизнес-ограничения

- Бюджет: только газ сети Sepolia (бесплатно для разработки)
- Срок первого релиза (MVP): 4 недели
- Поддержка только тестовой сети (основная сеть — в будущем)
- Команда: 1 разработчик

## 4. Бизнес-риски

| Риск | Вероятность | Влияние | Меры снижения |
|------|-------------|---------|----------------|
| Высокая стоимость газа | Средняя | Высокое | Использовать events вместо storage |
| Низкое внедрение вузами | Высокая | Среднее | Сделать простой интерфейс и документацию |
| Потера доступа к MetaMask | Низкая | Среднее | Дать инструкции по восстановлению |
