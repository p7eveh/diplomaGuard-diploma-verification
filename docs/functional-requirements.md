# Функциональные требования и атрибуты качества DiplomaGuard

## 1. Функциональные требования

| ID | Требование | Приоритет |
|----|------------|-----------|
| FR-01 | Система должна позволять администратору вуза выпускать диплом через смарт-контракт | Высокий |
| FR-02 | Система должна генерировать уникальный ID для каждого диплома | Высокий |
| FR-03 | Система должна позволять любому пользователю проверять диплом по ID без регистрации | Высокий |
| FR-04 | Система должна возвращать результат проверки (действителен/не действителен) | Высокий |
| FR-05 | Система должна показывать метаданные диплома (ФИО, специальность, вуз) при проверке | Средний |
| FR-06 | Только авторизованные адреса (вузы) могут выпускать дипломы | Высокий |
| FR-07 | Система должна логировать все выпущенные дипломы в событии смарт-контракта | Средний |

## 2. Атрибуты качества (нефункциональные требования)

| ID | Атрибут | Требование | Метрика |
|----|---------|------------|---------|
| QR-01 | Производительность | Время верификации | < 15 секунд |
| QR-02 | Производительность | API проверки | < 2 секунд (без учёта блокчейна) |
| QR-03 | Доступность | Доступность системы | 99.9% (кроме сбоев Ethereum) |
| QR-04 | Безопасность | Только админ может выпускать | Ролевая модель в контракте |
| QR-05 | Безопасность | Невозможность подделки | Блокчейн гарантирует неизменяемость |
| QR-06 | Юзабилити | Проверка без регистрации | 100% проверок без входа |
| QR-07 | Экономичность | Стоимость выпуска | < $0.50 |
| QR-08 | Масштабируемость | Количество дипломов | Не ограничено (только газом) |

## 3. Ограничения

| ID | Ограничение | Значение |
|----|-------------|----------|
| CO-01 | Блокчейн | Только Ethereum Sepolia (тестовая сеть) |
| CO-02 | Язык смарт-контракта | Solidity 0.8.x |
| CO-03 | Фронтенд | HTML/JS + ethers.js |
| CO-04 | Кошелёк | MetaMask |
| CO-05 | Хранение метаданных | Публичный JSON (GitHub Gist или аналоги) |
| CO-06 | Команда | 1 человек |
